NovelAI 이미지 생성 연동

Dynamic Sprite Emotion에 NAI 직접 연동 기능이 추가되었습니다.
캐릭터 표정 스프라이트를 AI로 자동 생성하고 즉시 등록할 수 있습니다.

DEMO — 표정 입력 → 생성 → 리스트 등록 → 배경 투명화

시작하기

  1. NAI API 키 발급

    novelai.net 로그인 → 우측 상단 프로필 → Account → Get Persistent API Token에서 키를 복사합니다.

  2. 확장 설정 열기

    SillyTavern 상단 Extensions 아이콘 → Dynamic Sprite Emotion → 캐릭터 선택 후 NAI 이미지 생성 섹션으로 이동합니다.

  3. API 키 입력 & 캐릭터 프롬프트 설정

    API 키를 붙여넣고, 캐릭터 베이스 프롬프트에 캐릭터의 외형을 입력합니다. (예: 1girl, silver hair, blue eyes, school uniform)

  4. 표정 생성

    단일 생성 칸에 표정 라벨명을 입력하고 🎨 생성을 클릭합니다. 생성된 이미지는 자동으로 스프라이트 리스트에 등록됩니다.

주요 기능

단일 생성 & 표정 가중치

단일 생성 행은 가중치 칸과 라벨명 칸 두 개로 구성됩니다.

[ 2 ] [ hurt ] [ 🎨 생성 ]

입력한 라벨이 기본 표정 목록(smile, sad, angry…)에 없으면 NAI의 표정 태그 형식으로 자동 변환해 전송합니다:

가중치::라벨명:: 예) 2::hurt::

기본 표정 목록 (smile, sad, angry, surprised, fear, disgust, neutral, happy, embarrassed, crying)은 최적화된 프롬프트로 전송됩니다. 그 외 라벨은 가중치 태그 방식으로 전송됩니다.

가중치 범위는 0.1 ~ 10이며 기본값은 2입니다. 값이 높을수록 해당 표정이 더 강하게 반영됩니다.

배경 제거 상세

배경 제거 ON 상태에서 이미지를 생성하면 흰 배경이 자동으로 투명화됩니다.

내부적으로 BFS(너비 우선 탐색) 플러드 필 방식을 사용합니다. 이미지의 테두리 픽셀에서 시작해 연결된 흰 픽셀만 제거하므로, 캐릭터가 흰 옷을 입고 있어도 옷은 그대로 유지됩니다.

임계값을 낮추면(예: 220) 더 좁은 범위만 흰색으로 처리합니다. 배경이 완전히 제거되지 않는다면 값을 높이고, 옷이 같이 지워진다면 값을 낮추세요.

시드 고정 & 일관성 유지

시드 고정을 ON하면 모든 생성에 동일한 시드 번호가 사용됩니다. 같은 시드와 같은 베이스 프롬프트를 쓰면 포즈와 얼굴 형태가 비슷하게 유지되어 표정 세트의 통일감을 높일 수 있습니다.

🎲 버튼을 누르면 랜덤 시드를 즉시 생성해 입력 칸에 채워줍니다.

시드가 같아도 Steps, CFG, 프롬프트가 다르면 결과물이 달라질 수 있습니다. 표정 세트를 처음 생성할 때 마음에 드는 시드를 기록해두고 고정하는 것을 권장합니다.

주의사항

  1. API 키는 저장 주의

    API 키는 SillyTavern 설정 파일에 저장됩니다. 설정 파일을 공유하거나 업로드할 때 키가 포함되지 않도록 주의하세요.

  2. NAI 구독 필요

    이미지 생성 API는 활성화된 NAI 구독이 있어야 사용할 수 있습니다. Anlas(포인트)가 소비됩니다.

  3. 동시 생성 제한

    NAI는 계정당 동시 생성 1건을 허용합니다. 다른 탭에서 생성 중이면 429 오류가 발생하며, 확장이 자동으로 재시도합니다.
